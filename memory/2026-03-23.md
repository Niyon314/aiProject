## 🔧 问题修复

### Docker 构建失败修复

**问题**: Dockerfile 在编译阶段失败，CGO 启用但缺少 C 编译器

**错误日志**:
```
Dockerfile:17 | RUN CGO_ENABLED=1 GOOS=linux go build -a -o main ./cmd/main.go
exit status 128: fatal: could not read from remote repository
```

**根本原因**: 
- `golang:1.21-bookworm` 镜像没有预装 `gcc`
- SQLite 驱动 (`gorm.io/driver/sqlite`) 需要 CGO 编译 C 代码

**修复方案**:
1. 在 builder 阶段添加 `gcc` 和 `libc6-dev` 安装
2. 使用 `-installsuffix cgo` 确保 CGO 正确链接
3. 清理 apt 缓存减少镜像大小

**修改文件**: `/root/.openclaw/workspace/backend/docker/Dockerfile`
