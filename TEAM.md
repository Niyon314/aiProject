# 🦾 百变怪技术团队框架

## 团队架构

| 角色 | Agent | 职责 | GitHub 标签 |
|------|-------|------|------------|
| 🦾 主协调 | main (百变怪) | 任务分发、进度追踪、总结汇报 | - |
| 📦 产品经理 | agent-product | 需求分析、PRD、优先级 | product, requirements, prd |
| 🎨 前端开发 | agent-dev | Web/移动端界面 | frontend, ui, web |
| ☕ 后端 Java | agent-dev | API、业务逻辑、数据库 | backend, java, api |
| 🧪 测试工程师 | agent-qa | 单元测试、集成测试、QA | test, qa, bug |
| 🔧 运维工程师 | agent-ops | 部署、监控、CI/CD | ops, devops, deploy |
| 🔍 研究员 | agent-research | 技术调研、竞品分析 | research, spike |

## GitHub 仓库
- **Repo:** niyon/aiProject
- **触发方式:** Issue 标签自动分配给对应 Agent

## 协作流程

```
用户指令/需求
    │
    ▼
┌─────────────┐
│  百变怪 (主) │ ← 总结、分解任务、分发
└─────────────┘
    │
    ├──────────┬──────────┬──────────┬──────────┐
    ▼          ▼          ▼          ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│ 产品   │ │ 开发   │ │ 测试   │ │ 运维   │ │ 研究   │
└────────┘ └────────┘ └────────┘ └────────┘ └────────┘
    │          │          │          │          │
    └──────────┴──────────┴──────────┴──────────┘
                         │
                         ▼
                  ┌─────────────┐
                  │ GitHub Issues│
                  │ 共享会话     │
                  └─────────────┘
```

## 工作模式
- **半自动触发:** 关键节点需要用户确认
- **共享会话沟通:** Agent 之间通过会话消息协作
- **GitHub Issues 管理:** 任务状态追踪
