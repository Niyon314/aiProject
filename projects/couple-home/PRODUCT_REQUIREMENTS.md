## 👥 一、用户画像

### 目标用户

| 特征 | 描述 |
|------|------|
| **人群** | 20-35 岁情侣，同居或准备同居 |
| **关系状态** | 恋爱中、订婚、新婚 |
| **生活状态** | 共同居住，需要协调日常生活 |
| **技术接受度** | 高，习惯使用手机 App 管理生活 |

### 用户痛点

1. **吃饭安排难**
   - 每天纠结"吃什么"
   - 对方口味不了解
   - 外卖选择困难

2. **家务分工不清**
   - 谁洗碗、谁拖地经常争执
   - 家务完成度无法追踪

3. **账单混乱**
   - 谁付的钱记不清
   - AA 制计算麻烦

4. **仪式感缺失**
   - 纪念日容易忘记
   - 缺少共同回忆整理

5. **日程不同步**
   - 对方安排不清楚
   - 约会时间难协调

## 🎯 二、使用场景

### 场景 1: 今天吃什么？🍽️

**时间**: 每天上午 10 点、下午 4 点  
**触发**: 系统推送通知  
**流程**:
1. 双方收到"今天吃什么"投票
2. 各自选择喜欢的餐食（like/dislike/veto）
3. 系统自动匹配双方都接受的选项
4. 公布结果并推荐附近餐厅/外卖

### 场景 2: 家务分工 🧹

**时间**: 每周日晚上  
**触发**: 系统推送周家务清单  
**流程**:
1. 系统列出本周家务（洗碗、拖地、倒垃圾等）
2. 双方认领各自任务
3. 完成后拍照打卡
4. 积分排行榜，月末奖励

### 场景 3: 账单 AA 💰

**时间**: 随时  
**触发**: 用户主动记录  
**流程**:
1. 一方支付后拍照上传账单
2. 输入金额和类别（房租、水电、买菜等）
3. 系统自动计算 AA 金额
4. 月末生成账单报告

### 场景 4: 纪念日提醒 💕

**时间**: 纪念日前 7 天、3 天、1 天  
**触发**: 系统推送提醒  
**流程**:
1. 提前提醒准备礼物/安排约会
2. 当天推送祝福和回忆照片
3. 记录今年纪念日安排

### 场景 5: 照片墙 📸

**时间**: 随时  
**触发**: 用户主动上传  
**流程**:
1. 上传合照或生活瞬间
2. 自动按时间线整理
3. 可添加标签和心情
4. 生成月度/年度回忆视频

### 设计原则

1. **圆角设计**
   - 卡片圆角：16px
   - 按钮圆角：24px
   - 头像圆角：50%

2. **可爱元素**
   - 爱心装饰 💕
   - 浮动动画
   - Emoji 表情丰富

3. **字体**
   - 标题：圆润字体 (Nunito/Quicksand)
   - 正文：清晰易读 (Inter/思源黑体)

4. **交互**
   - 点击反馈 (缩放/颜色变化)
   - 加载动画 (爱心跳动)
   - 成功提示 (彩带动画)

## 🛠️ 五、技术栈建议

### 前端

| 技术 | 选择 | 理由 |
|------|------|------|
| **框架** | React + Vite | 快速开发、热更新 |
| **UI 库** | Tailwind CSS | 灵活定制可爱风 |
| **状态管理** | Zustand | 轻量、简单 |
| **请求库** | Axios | 成熟稳定 |
| **动画** | Framer Motion | 流畅可爱动画 |

### 后端

| 技术 | 选择 | 理由 |
|------|------|------|
| **框架** | NestJS | TypeScript、模块化 |
| **ORM** | Prisma | 类型安全、开发快 |
| **数据库** | PostgreSQL | 稳定、功能全 |
| **缓存** | Redis | 会话、实时数据 |
| **实时** | WebSocket | 通知推送 |

### 部署

| 服务 | 选择 | 理由 |
|------|------|------|
| **容器** | Docker | 环境一致 |
| **编排** | Docker Compose | 简单够用 |
| **CI/CD** | GitHub Actions | 免费、集成好 |
