## 📋 一、功能概述

### 1.1 产品目标

通过游戏化的方式解决家务分工问题，让做家务变得有趣，减少争执，增加成就感。

### 1.2 核心流程

```
每周日晚 8 点
       ↓
系统发布周家务清单
       ↓
双方认领各自任务
       ↓
完成后拍照打卡
       ↓
获得积分 + 排行榜
       ↓
月末兑换奖励
```

## 🎯 二、功能需求

### 2.1 家务任务管理

#### 任务类型

| 类型 | 频率 | 示例 | 基础积分 |
|------|------|------|----------|
| **每日** | 每天 | 洗碗、倒垃圾 | 10 分 |
| **每周** | 每周 | 拖地、洗衣服 | 30 分 |
| **每月** | 每月 | 大扫除、擦窗 | 100 分 |
| **临时** | 按需 | 修东西、搬家 | 协商 |

#### 数据结构

```typescript
interface Chore {
  id: string;                    // 任务 ID
  name: string;                  // 任务名称
  icon: string;                  // emoji 图标
  type: 'daily' | 'weekly' | 'monthly' | 'once';
  points: number;                // 积分
  assignee?: 'user' | 'partner'; // 认领人
  dueDate: string;               // 截止日期
  status: 'pending' | 'completed' | 'overdue';
  completedAt?: string;          // 完成时间
  proofPhoto?: string;           // 证明照片（可选）
  notes?: string;                // 备注
  createdAt: string;
  updatedAt: string;
}

interface ChoreTemplate {
  id: string;
  name: string;
  icon: string;
  type: 'daily' | 'weekly' | 'monthly';
  points: number;
  defaultAssignee?: 'user' | 'partner'; // 默认认领人
}
```

### 2.2 认领系统

#### 认领规则
1. **先到先得**: 谁先认领就是谁的
2. **可交换**: 双方同意可交换任务
3. **可求助**: 可请求对方帮忙（对方同意后可获得 50% 积分）

## 🛠️ 四、技术实现

### 4.1 后端 API

```go
// 家务任务
GET    /api/chores              // 获取任务列表
POST   /api/chores              // 创建任务
PUT    /api/chores/:id          // 更新任务
DELETE /api/chores/:id          // 删除任务

// 认领
POST   /api/chores/:id/claim    // 认领任务
POST   /api/chores/:id/swap     // 交换任务

// 打卡
POST   /api/chores/:id/complete // 完成打卡

// 统计
GET    /api/chores/stats        // 统计数据
GET    /api/chores/leaderboard  // 排行榜
GET    /api/chores/achievements // 成就列表
```

### 4.2 数据模型

```go
type Chore struct {
    ID          string     `json:"id"`
    Name        string     `json:"name"`
    Icon        string     `json:"icon"`
    Type        string     `json:"type"` // daily/weekly/monthly/once
    Points      int        `json:"points"`
    Assignee    string     `json:"assignee,omitempty"` // user/partner
    DueDate     time.Time  `json:"dueDate"`
    Status      string     `json:"status"` // pending/completed/overdue
    CompletedAt *time.Time `json:"completedAt,omitempty"`
    ProofPhoto  string     `json:"proofPhoto,omitempty"`
    Notes       string     `json:"notes,omitempty"`
    CreatedAt   time.Time  `json:"createdAt"`
    UpdatedAt   time.Time  `json:"updatedAt"`
}

type UserStats struct {
    UserID          string `json:"userId"`
    TotalPoints     int    `json:"totalPoints"`
    CompletedTasks  int    `json:"completedTasks"`
    OnTimeRate      float64 `json:"onTimeRate"`
    CurrentStreak   int    `json:"currentStreak"`
    LongestStreak   int    `json:"longestStreak"`
}
```
