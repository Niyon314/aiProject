## 📋 一、功能概述

### 1.1 产品目标

解决情侣每天"吃什么"的难题，通过投票机制让双方共同参与决策，增加互动乐趣。

### 1.2 核心流程

```
上午 10 点 / 下午 4 点
       ↓
系统推送通知
       ↓
双方进入投票页面
       ↓
各自选择 like/dislike/veto
       ↓
系统匹配双方都接受的选项
       ↓
公布结果 + 推荐餐厅/外卖
```

## 🎯 二、功能需求

### 2.1 投票系统

#### 功能描述
- 每天生成 3-5 个餐食选项（基于菜谱库）
- 双方独立投票，互不可见
- 支持三种投票类型：
  - 👍 **Like**: 想吃
  - 👎 **Dislike**: 不想吃
  - ❌ **Veto**: 强烈反对（ veto 权优先）

#### 数据结构

```typescript
interface MealVote {
  id: string;                    // 投票 ID
  date: string;                  // 投票日期 (YYYY-MM-DD)
  mealType: 'lunch' | 'dinner';  // 餐食类型
  options: MealOption[];         // 选项列表
  userVote?: UserVote;           // 当前用户投票
  partnerVote?: UserVote;        // 伴侣投票（仅自己投票后可见）
  result?: MealOption;           // 匹配结果
  status: 'pending' | 'voted' | 'completed';
  createdAt: string;
  updatedAt: string;
}

interface MealOption {
  id: string;           // 菜谱 ID
  name: string;         // 菜谱名称
  icon: string;         // emoji 图标
  cookTime: number;     // 烹饪时间 (分钟)
  difficulty: string;   // 难度
  cost: number;         // 预估成本
  tags: string[];       // 标签
}

interface UserVote {
  optionId: string;     // 选择的选项 ID
  type: 'like' | 'dislike' | 'veto';
  timestamp: string;
}
```

#### 匹配规则

1. **双方 Like 同一选项** → 直接匹配 ✅
2. **一方 Like，一方 Dislike** → 不匹配
3. **任意一方 Veto** → 该选项淘汰 ❌
4. **无共同 Like** → 推荐双方都不反对的最高分选项

### 2.2 通知系统

#### 推送时间
- **午餐投票**: 上午 10:00
- **晚餐投票**: 下午 16:00

#### 推送文案
```
🍽️ 今天吃什么？

亲爱的，该选午餐啦~
已经有 3 个美味选项等你投票哦！

[立即投票] [稍后再说]
```

#### 提醒机制
- 投票发出后 30 分钟未投票 → 提醒一次
- 投票截止前 1 小时未投票 → 再次提醒

#### 公布时机
- 双方都完成投票后立即公布
